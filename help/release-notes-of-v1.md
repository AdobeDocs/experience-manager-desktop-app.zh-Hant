[!DNL Adobe Experience Manager] 桌面應用程式與下列作業系統相容：

* MacOS X 10.10或更高版本，帶有最新的錯誤修復。

* Windows 10帶有最新的Service Pack和錯誤修復程式。

>[!NOTE]
>
>不再支援Windows 7。 請參閱 [關於Windows 7 EOL的文章](https://support.microsoft.com/en-us/help/4057281/windows-7-support-ended-on-january-14-2020)。

Adobe強烈建議使用最新版AEM的案頭應用程式來使用最新的功能、最新的錯誤修復程式以及盡可能最好的效能。

您計畫在AEM本地電腦上安裝的案頭應用版本需要特定的AEM伺服器版本/附加的伺服器端元件（服務包、熱修復程式或功能包）。 在首次連AEM接到伺服器之前，請確保已正確配置伺服器。 如果需要幫助，請與管理員AEM聯繫。

查看 [詳細相容性矩陣](#compatibilitymatrix) 以評估您的設定的先決條件。

## 案頭應用1.10版中的新增功能 {#what-s-new-in-aem-desktop-app}

案頭AEM應用1.10側重於改進用戶在大量上載、後台操作資訊以及使用連結檔案開啟資產時的最佳體驗(如InDesign)方面的體驗。

>[!NOTE]
>
>如果您使用的是macOS10.15.4或更新版本，請至少使用1.10.0.6版的應用。 此修補程式版本符合 [Apple公證要求](https://developer.apple.com/news/?id=04102019a)。

**本地編輯/簽出**:可以在狀態窗口中禁用保存到資產的更改的自動上載。 這樣，用戶就可以繼續處理檔案並保存更改，然後，當它們準備好時，就決定上載所有更改。

**簡化資產狀態窗口**。 狀態窗口已簡化。 的 [!UICONTROL Uploads] 頁籤現在顯示單個資產以及資料夾或批量上載。 已刪除上一個「批量上載」頁籤。

**指示批量上載的應用程式表徵圖**。 應用程式表徵圖通過顯示「傳輸」覆蓋來指示正在進行批量上載。

**更新衝突通知**。 當應用程式在嘗試更新資產時檢測到衝突時，它將顯示通知，以便用戶無需監視狀態窗口即可查看。 當應用程式啟動時，它將檢查所有衝突，以便用戶能夠解決這些衝突。

**更好地處理連接損失**。 如果連接丟失，將暫停批量上載，用戶以後將能夠恢復。 A [!UICONTROL Retry] 選項可用於重試單個檔案的上載失敗。

### 自案頭應AEM用1.9以來提供的增強功能 {#Enhancements-Available-Since-AEM-Desktop-App-19x}

[!DNL Adobe Experience Manager] desktop app 1.9.1是一個修補程式版本，用於解決資產簽出和將檔案從網路共用複製到本地目錄等幾個關鍵客戶問題。

* 由一個用戶簽出的資產不應可供其他用戶修改(CQ-4246009)

* 當用戶資料夾位於單獨的磁碟分區上時，支援從映射資料夾複製到本地資料夾(CQ-4243978)

案頭應AEM用1.9側重於改進用戶在大量上載、後台操作資訊以及使用連結檔案開啟資產時的優化體驗(如InDesign)。

**可恢復上載**
對於上載，特別是大型檔案，有一個選項可以在新的「資產狀態」窗口中暫停/恢復這些檔案。

**改進的資產狀態窗口**
改進的資產狀態窗口提供了以下有關資產的資訊。

[!UICONTROL Changes]

* 顯示當前排隊的更改。

* 顯示正在進行的上載，包括進度欄、傳輸速率、總檔案大小和到目前為止傳輸的大小。

* 已完成上載，顯示總傳輸速率和最終速率。

* 失敗的上載將顯示一條錯誤消息並傳輸資訊（如果可用）。

* 失敗3次的上載將顯示錯誤消息。

* 用用戶可按一下的表徵圖顯示的衝突檔案。 按一下該表徵圖將顯示一個對話框，其中包含說明和兩個選項：

   * [!UICONTROL Keep Mine] 立即將檔案上載到伺服器。

   * [!UICONTROL Overwrite Mine] 立即刪除本地檔案並從伺服器下載新副本。

[!UICONTROL Downloads]

* 顯示正在進行的下載，包括到目前為止傳輸的速率和大小。

* 已完成的下載顯示為已傳輸的總數、最終速率和一個表徵圖，按一下後將開啟該檔案（僅適用於單個檔案）。

* 下載失敗，顯示錯誤消息並傳輸資訊（如果可用）。

* 頁腳顯示下載的檔案總數和平均傳輸速率。

* 如果用戶選擇從 [!DNL Experience Manager Assets] Web介面，它們將被分組在一起。 例如，myasset.jpeg和4個其他檔案。

* 下載InDesign文檔(包括儲存在AEM Assets的連結資產)時，案頭應用將首先下載所有連結資產，然後再開啟 [!UICONTROL Adobe InDesign] 並指明連結資產的下載。 例如，24人中有5人。

[!UICONTROL Bulk Uploads]

通過 [!UICONTROL Create] > [!UICONTROL Upload Folder] 在AEMWeb UI中，或在案頭應用上下文菜單中的Finder或Explorer中複製和選擇「貼上資產」，將觸發此對話框的使用。

* 顯示正在進行的上載，包括進度欄和當前正在傳輸的檔案的名稱。

* 正在進行的上載包含一個表徵圖，該表徵圖將在按一下時取消上載。 傳輸將在當前傳輸檔案完成後停止。

* 失敗的傳輸進程將顯示並顯示錯誤消息（僅當整個傳輸失敗時）。

* 如果單個檔案無法傳輸，則該檔案將顯示為錯誤。 否則，單個檔案將不會在頁籤*上顯示整個上載的單個條目。

**用於指示後台操作狀態的表徵圖**

應用程式表徵圖將指示後台操作的狀態，以便為用戶提供更好的視覺提示。 例如，當應用程式未連接到圖AEM標時，表徵圖將呈灰色顯示，當存在活動上載時，它將顯示「同步」覆蓋等。

**連結資產的預取**

為了在處理包含儲存在中的連結資產的InDesign文檔時改善用戶體驗AEM，案頭應用程式將嘗試在下載和開啟InDesign文檔之前將這些連結檔案預取到本地快取。 這樣，用戶就可以在本地使用連結檔案，在InDesign（在「連結」面板中）中訪問這些檔案時不必再等待。
請注意，預取僅在識別伺服器端AEM的連結時才起作用。 具有已識別連結的資產將在InDesign資產的「屬性」視圖中列出「引用」清單。

### 自案頭應AEM用1.5以來提供的增強功能 {#Enhancements-Available-Since-AEM-Desktop-App-15}

**用於MacOS X的1.5.1.5版：** 1.5.1.5版提供了以下好處：

* 新功能和增強功能：將「複製/貼上」功能添加到Finder整合中，以允許從案頭直接AEM傳輸到(CQ-4208158)。

* 錯誤修復：

   * 修復錯誤43問題，在資產更名期間(CQ-4207900)有時發生。

   * 從中的時間線還原到舊版本AEM不會更新Finder中的資產(CQ-4205194)。

   * 瀏覽大型嵌套目錄時案頭應用崩潰(CQ-4208539)。

   * 案頭應用裝載點現在為/Volumes/DAM，因此它對所有用戶(CQ-4208159)是一致的。

   * 首次將檔案放入InDesign中會啟動更新警告(CQ-4207454)。

連結警告注意事項：Creative Cloud應用程式(如InDesign)對項目放置時上次修改的時間進行「快照」。 如果日期稍後更改，Adobe Creative Cloud應用將報告連結已過期。 本文通過以下幾種方式報告：

* 當Adobe Creative Cloud應用啟動時，它將顯示一個對話框，通知用戶連結的資產已過期，並提示用戶採取操作。

* 如果Adobe Creative Cloud應用已在運行，它將在連結的資產上顯示一個黃色三角警告表徵圖。

此行為與本地磁碟上的資產和AEMDesktop掛載目錄中的資產相同，但有以下例外：

* 如果已放置的資產由其他用戶修改，則警告表徵圖將首次顯示其他用戶開啟包含已放置資產的文檔的時間。 僅當已本地快取已放置的資產時，才會發生這種情況。

* 如果用戶通過案頭的AEM掛載目錄修改已置資產，然後清除其本地快取，則已置資產將報告為過期。

這兩種情況都是預期的，是案頭「延遲同步」體系結構的副AEM作用。

**用於MacOS X和Windows的1.5.0.x版：** 此版本的AEM案頭應用具有以下優點：

* 更好地穩定和抵御網路問題。

   * 更穩定的AEM Assets資料夾映射(CQ-103276、CQ-4204669、CQ-4203957)。

   * 對大於2 GB的大檔案的下載/上載處理得到改進(CQ-4206438)。

   * 在Finder中移動或更名較多檔案(CQ-4204640)時已修復「錯誤36」。

* 與伺服器AEM(CQ-4204974、CQ-100903)的網路通信優化。

* 在Creative Cloud應用中開啟、放置和AEM保存資產的可靠性得到提高(CQ-4203968、CQ-4205511、CQ-103543、CQ-4207141、CQ-90980)。

* 提高了支援性：選項以清除快取(CQ-4202541)，輕鬆訪問日誌(CQ-4202340、CQ-4204673)。

* 其他修復：
   * 更好地支援名稱/非英語設定中具有日文字元的資產和資料夾(CQ-4195433、CQ-4205793、CQ-4199446)。

   * 更好地處理使用SSL登錄(CQ-4200217)。

   * 更可靠的共用安裝(CQ-4200793)。

## 更改清單 {#list-of-changes}

### Mac山 {#mount-point-on-mac}

自MacOS10.12(Sierra)以來，Apple已將用於裝載網路驅動器和設備的/Volumes資料夾的權限更改為限制性更強。 在那裡建立新的裝載點需要管理權限。 此問題已在MacOS10.12.5解決。

由AEM於案頭應用應為在本地電腦上沒有管理員權限的用戶運行，因此AEM Assets儲存庫的掛載點在1.4和1.5中更改為MacOS上用戶本地資料夾(CQ-104183)中的DAM子資料夾。

由於/Volumes資料夾不再需要管理權限，因此此更改在1.5.1中恢復。這還使得可以在MacOS用戶之間共用已AEM將資產置於InDesign的文檔。

### 協定更改（自v1.3） {#protocol-change-since}

* MacOS X:
   * 用於OS X案頭整合的本地網路驅動器協定從WebDAV更改為SMB1。
   * 隨桌AEM面應用裝載的儲存庫在Finder中將顯示為「smb」網路驅動器，而不是WebDAV驅動器。
* Windows:
   * Windows案頭整合的本地網路驅動器協定；將AEM作為WebDAV共用裝載。
* 對於兩個平台(Windows和Mac):
   * 訪問/下載資產和上載更改的協定AEM已更改為本機AEM協定，該協定是基於HTTP的REST風格協定。 它提供了對網路操作的更大控制，並且與網路基礎架構更相容。

>[!NOTE]
>
>在MacOS X上，本地網路驅動器協定從WebDAV更改為SMB1會導致儲存庫中同一資產的不同本地路徑。 這可能會通過「放置」命令影響到放置在Adobe Creative Cloud應用程式中的檔案的連結。 查看 [使用桌AEM面應用](use-app-v1.md) 的子菜單。

### 檔案處理（自1.3以來） {#file-handling-since}

* 資料夾在預定義延遲後（當前為30秒）自動更新。
* 其他用戶簽出的檔案被標籤為只讀。
* 檔案將分兩個階段保存到通過案頭應用程式裝載的網路驅動器位置。
* 在第一階段中，檔案被本地保存。 這樣，保存檔案的用戶無需等待檔案完全傳輸到AEM，並且一旦保存檔案就可以恢復工作。
* 在第二階段，案頭應用在預定義AEM的延遲後（例如30s）將更新的檔案上載到伺服器。 此操作在後台進行。 使用 **顯示後台檔案同步狀態** 的子菜單。

## 重要通知 {#important-notices}

**資料夾上載。** 建議使用新的「資料夾上載」功能將較大的分層資料夾上載到AEM，而不是使用從Finder/Explorer級別將拷貝/拖放到AEM已裝載的儲存庫中。 當使用資料夾上載功能時，案頭應用會直接AEM與其通信，從而更好地控制整個過程。

**保持AEM會話可用。** 案頭應AEM用依賴於開啟給AEM Assets伺服器的會話，以確保正確操作。 對於每天使用案頭應用的用戶，建議在其工作日結束時卸下AEM Assets以強制註銷，然後在早上「裝載AEM Assets」以確保他們已登錄並且網路共用可以正常運行。

**在Finder中關閉「表徵圖預覽」。** 要使用Finder對大型資料夾進行效能瀏覽，特別是網路連接性較差時，請確保同時關閉「表徵圖」和「表徵圖預覽」。 否則，Finder將開始下載資料夾中的每項資產，以生成小的預覽，這可能導致效能差和高頻寬利用率(CQ-4219779)

* 在查找器中，轉到AEM Assets共用網路資料夾
* 按一下右鍵DAM安裝點
* 選擇顯示視圖選項
* 取消選擇「顯示表徵圖預覽」
* 按一下「用作預設值」

**連接到新伺服器時清AEM除快取。** 如果案頭應用連接到AEM具有相同URL的另一台伺服器，則不會自動清除快取。 手動清除快取，以確保正確操作。 請注意，在同一URL上運行時，AEM當可以替換安裝(CQ-4216982)時，通常會在測試中出現這種情況

**使用CA簽名的SSL證書。** 請注意，AEM通過HTTPS安全連接連接到時，案頭應用不支援自簽AEM名的SSL證書。 伺服器上需要CA簽名的證書才能進行此類連接。 (CQ-87941)

## 已知問題 {#known-issues}

* 一般:
   * 伺服器URL需要指向沒有路徑的伺服器(例如， `http://server`。 `https://server`。 `http://server:port`或 `https://server:port`)。 不支援/content/dam以外的上下文路徑和子資料夾(CQ-89343、CQ-87272)
* 檔案名/本地化：
   * 未正確處理帶保留字元的檔案和資料夾名稱。 確保使用符合要求的檔案AEM和資料夾名稱(CQ-93361、CQ-93308、CQ-89276、CQ-4217183)
   * 某些應用程式(如Adobe Illustrator)可能會建立中不支援的名稱的AEM檔案。 例如，添加 `Converted` 轉換檔案後，檔案將停止上載(CQ-4216985)
   * 具有國際名稱的資產可能每隔幾秒鐘就會出現並消失
* 簽入和簽出：
   * 一個用戶簽出的資產無法通過觸摸UI的「開啟」操作或直接在案頭上為另一用戶開啟。 某些應用程式可能報告它已鎖定，但在嘗試開啟時也已損壞甚至掛起。 (CQ-4199234)
   * 同時由多個用戶更改檔案可能會導致某些修改丟失。 解決方法是使用簽入和簽出功能來防止多個用戶更改同一檔案(CQ-97035)
   * 某些應用程式不正確支援只讀標誌，這允許用戶保存由其他用戶簽出的檔案。 在其他用戶簽入檔案之前，不會傳輸修改的檔案。 兩種修改都可AEM作為資產的不同版本(CQ-89551、CQ-87572、CQ-89615)使用
   * Finder中獨立報告已檢出和只讀狀態。 當用戶簽出資產時，這會生成2個鎖定表徵圖(CQ-89507)
* Finder整合：
   * 在拖放大型檔案時，當檔案在後台傳輸時，Finder可能會超時。 這將導致 `Error - 36`。 解決方法是再次拖放或開啟資產(CQ-4219628)
   * 手動資料夾重新載入並不總能奏效。 解決方法：等待30s以自動更新資料夾。 (CQ-97389)
   * 更多資產資訊……僅限於單個檔案選擇(CQ-89542、CQ-87656)
   * 在AEM Assets開門……僅限於單個檔案和資料夾選擇(CQ-83382)
   * 更名無副檔名的資產時出錯(CQ-4218971)
* 複製/貼上功能：當未將任何資產複製到剪貼簿時，貼上可用
* Windows:
   * 只有NTFS上完全支援具有備用資料流(ADS)的檔案。 將這些檔案複製到案頭應用提供的WebDAV共用中將導致警告對話框，警告用戶該檔案具有無法複製到新位置的屬性。 這通常沒問題，因為這些屬性僅與用戶案頭上的特定應用程式相關，與實際檔案內容(CQ-103770)(Win)無關
   * Windows上的案頭應用程式需要由將要使用它的用戶安裝(CQ-4216389)(win)
   * 選擇 [!UICONTROL Retry] 選項在斷開連接後恢復批處理上載後在某些情況下失敗上載(CQ-4251884)(Win)
