## 系統需求和先決條件 {#system-requirements-and-prerequisites}

AEM Desktop與下列作業系統相容：

* Mac OS X 10.10 或更新版本，提供最新的錯誤修正。
* Windows 7 和 Windows 10，提供最新的服務套件和錯誤修正。

Adobe強烈建議使用最新版AEM案頭應用程式，以取得最新功能、最新錯誤修正以及最佳效能。

您打算安裝在本機電腦上的AEM案頭應用程式版本需要特定的AEM伺服器版本／其他伺服器端元件（服務套件、Hotfix或功能套件）。 在您第一次連線至AEM伺服器之前，請確定AEM伺服器已正確設定。 如果您需要協助，請連絡您的AEM管理員。

請參 [閱本檔案結尾的詳細相容性表](#compatibilitymatrix) ，以評估您設定的必要條件。

## What&#39;s New in AEM desktop app 1.10 {#what-s-new-in-aem-desktop-app}

AEM案頭應用程式1.10著重在改善大型上傳的使用者體驗、背景作業的相關資訊，以及在使用連結檔案（例如InDesign）開啟資產時的最佳化體驗。

>[!NOTE]
>
>如果您使用macOS 10.15.4或更新版本，請至少使用應用程式1.10.0.6版。 此修補程式發行符合 [Apple公證要求](https://developer.apple.com/news/?id=04102019a)。

**本機編輯／結帳**: 自動上傳儲存至資產的變更可在狀態視窗中停用。 如此，使用者就可繼續處理檔案並儲存變更，然後當變更準備就緒時，決定上傳所有變更。

**簡化的資產狀態視窗**。 狀態窗口已簡化。 標籤 [!UICONTROL Uploads] 現在會顯示個別資產，以及資料夾或大量上傳。 先前的「大量上傳」索引標籤已移除。

**「應用程式」圖示，以指出大量上傳**。 應用程式圖示會顯示「轉移」覆蓋，以指出大量上傳正在進行中。

**更新衝突通知**。 當應用程式在嘗試更新資產時偵測到衝突時，會顯示通知，讓使用者不必監視狀態視窗即可檢視。 當應用程式啟動時，會檢查所有衝突，讓使用者可以解決這些衝突。

**更好地處理連接損失**。 如果連線中斷，則會暫停大量上傳，而使用者稍後將可繼續。 「重試」按鈕可用於重試個別檔案的上傳失敗。

## 舊版的增強功能 {#enhancements-in-the-previous-versions}

此版本可延伸並取代舊版Experience Manager案頭應用程式，後者提供下列主要增強功能：

* **1.9/1.9.1版**: 可恢復的上傳、改良的狀態視窗、指示應用程式／連線狀態的應用程式圖示、InDesign檔案的連結資產預取。

* **1.8版**: 更好地控制使用者的快取大小、改善Windows上SAML/SSO的登入體驗、支援Mac上的。pac網路proxy，以及客戶回報的問題。

* **1.7版**: 改善穩定性和快取邏輯、更佳的網路代理支援，以及解除安裝後清除內部檔案的功能。

* **1.6版**: 針對各種AEM安全性組態和應用程式穩定性與效能的登入程式改進。

* **1.5版**: 應用程式穩定性和針對各種網路問題的恢復能力，以及更好的支援性。

* **1.4版**: 能夠使用進度監控在後台上載分層資料夾。

* **1.3版**: 存取檔案和儲存AEM變更的效能增強與穩定性，尤其是從Creative Cloud案頭應用程式（例如InDesign、Illustrator或Photoshop）儲存變更。 它旨在在處理檔案時，為使用者提供更本端的案頭樣式體驗，同時在背景處理網路資料傳輸作業。

* 一個使用者簽出的資產不應可供其他使用者修改(CQ-4246009)

* 當使用者資料夾位於個別磁碟分區時，支援將資料夾從映射資料夾複製到本機資料夾(CQ-4243978)

AEM案頭應用程式1.9著重於改善大型上傳的使用者體驗、背景作業的相關資訊，以及在使用連結檔案（例如InDesign）開啟資產時的最佳化體驗。

**可恢復的上**&#x200B;載對於上載，尤其是大型檔案的上載，有一個選項可在新的「資產狀態」視窗中暫停／繼續。

**改進的資產狀態視**&#x200B;窗改進的資產狀態視窗提供下列資產相關資訊。

[!UICONTROL Changes]

* 顯示當前排隊的更改。

* 顯示進行中的上載，包括進度列、傳輸速率、總檔案大小和目前傳輸的大小。

* 已完成的上傳會以總傳輸和最終速率顯示。

* 失敗的上傳會顯示錯誤訊息，並傳輸資訊（如果有的話）。

* 失敗3次的上傳會顯示錯誤訊息。

* 使用使用者可點按的圖示顯示的檔案衝突。 按一下圖示會顯示對話方塊，其中包含說明和兩個選項：

   * [!UICONTROL Keep Mine] 立即將檔案上傳至伺服器。

   * [!UICONTROL Overwrite Mine] 立即刪除本機檔案，並從伺服器下載新副本。

[!UICONTROL Downloads]

* 顯示進行中的下載，包括到目前為止已傳輸的傳輸速率和大小。

* 已完成的下載會顯示總傳輸量、最終速率，以及在點按時開啟檔案的圖示（僅適用於單一檔案）。

* 失敗的下載會顯示錯誤訊息，並傳輸資訊（如果有的話）。

* 頁尾顯示下載的檔案總數和平均傳輸率。

* 如果使用者選擇從Experience Manager Assets Web UI開啟或編輯多個檔案，則會將檔案分組在一起。 例如，myasset.jpeg和4個以上的檔案。

* 當下載InDesign檔案（包括儲存在AEM Assets中的連結資產）時，案頭應用程式會先下載所有連結的資產，然後再開啟檔案並指出 [!UICONTROL Adobe InDesign] 連結資產的下載。 例如，24個中的5個。

[!UICONTROL Bulk Uploads]

在AEM網頁UI中透過 [!UICONTROL Create][!UICONTROL Upload Folder] >上傳大型檔案夾階層，或在Finder或案頭應用程式內容選單中複製並選取「貼上資產」，將會觸發此對話方塊的使用。

* 顯示進行中的上載，包括進度列和當前正在傳輸的檔案名。

* 進行中的上載包含一個圖示，在點按時會取消上載。 傳輸將在當前傳輸檔案完成後停止。

* 失敗的傳輸進程會顯示一條錯誤消息（僅當整個傳輸失敗時）。

* 如果個別檔案無法傳輸，則會在標籤上顯示為錯誤。 否則，標籤*上不會只顯示整個上載的單一項目。

**用於指示後台操作狀態的表徵圖**

應用程式圖示會指出背景作業的狀態，為使用者提供更佳的視覺提示。 例如，當應用程式未連線至AEM時，圖示將會變灰，當有作用中的上傳時，它會顯示「同步」覆蓋等。

**連結資產的預取**

為改善使用包含儲存在AEM中之連結資產的InDesign檔案時的使用體驗，案頭應用程式會先嘗試將這些連結檔案預先擷取至本機快取，然後再下載並開啟InDesign檔案。 如此，使用者就可在本機取得連結的檔案，而且在InDesign（在「連結」面板中）中存取這些檔案時，不需要等待更久。
請注意，預取只有在AEM識別伺服器端的連結時才能運作。 具有已辨識連結的資產會在InDesign資產的「屬性」檢視中列出「參考」清單。

   * 案頭應用程式在瀏覽大型巢狀目錄時當機(CQ-4208539)。

   * 案頭應用程式裝載點現在是/Volumes/DAM，因此對所有使用者而言都是一致的(CQ-4208159)。

連結警告注意事項： Creative Cloud應用程式（例如InDesign）會拍攝項目放置時上次修改時間的「快照」。 如果日期稍後變更，Adobe Creative Cloud應用程式會報告連結已過時。 以下幾種方式報告了此問題：

* 當Adobe Creative Cloud應用程式啟動時，會顯示對話方塊，通知使用者連結的資產已過時，並提示使用者採取動作。

* 如果Adobe Creative Cloud應用程式已在執行中，則會在連結的資產上顯示黃色的三角形警告圖示。

本端磁碟上的資產和AEM Desktop掛載目錄中的資產的行為相同，但有下列例外：

* 如果已置入的資產由不同的使用者修改，則警告圖示會在其他使用者第一次開啟包含已置入資產的檔案時顯示。 只有已在本機快取已置入的資產時，才會發生此情況。

* 如果使用者透過AEM案頭的已載入目錄修改已置入的資產，然後清除其本機快取，則已置入的資產將會報告為過期。

這兩種情況都符合預期，是AEM Desktop「延遲同步」架構的副作用。

## 變更清單 {#list-of-changes}

### Mac上的裝載點 {#mount-point-on-mac}

自從MacOS 10.12(Sierra)以來，Apple已將用於安裝網路磁碟機和裝置的/Volumes檔案夾的權限變更為限制性。 在此處建立新的裝載點需要管理權限。 此問題已在MacOS 10.12.5中修正。

由於AEM案頭應用程式應針對在本機電腦上沒有管理員權限的使用者執行，因此AEM Assets存放庫的裝載點已在1.4和1.5中變更為MacOS使用者本機資料夾中的DAM子資料夾(CQ-104183)。

由於/Volumes資料夾不再需要管理權限，因此在1.5.1中恢復了此更改。 這也可讓MacOS使用者共用已放置AEM資產的InDesign檔案。

### 通訊協定變更（自v1.3起） {#protocol-change-since}

* Mac OS X:
   * OS X案頭整合的本地網路驅動器協定從WebDAV更改為SMB1。
   * 隨案頭應用程式載入的AEM存放庫在Finder中會顯示為「smb」網路磁碟機，而非WebDAV磁碟機。
* Windows:
   * Windows案頭整合的本端網路驅動器通訊協定仍維持； AEM將會以WebDAV共用方式載入。
* 適用於兩種平台（Windows和Mac）:
   * 存取／下載資產和上傳AEM變更的通訊協定已變更為原生AEM通訊協定，此為以HTTP為基礎的REST風格通訊協定。 它提供對網路操作的更大控制，並且與網路基礎架構更相容。

>[!NOTE]
>
>在Mac OS X上，本地網路驅動器協定從WebDAV更改為SMB1會導致儲存庫中同一資產的本地路徑不同。 這可能會影響透過「置入」命令置入Adobe Creative Cloud應用程式的檔案連結。 如需詳細 [資訊，請參閱「使用AEM案頭應用程式](use-app-v1.md) 」。

### 檔案處理（自1.3起） {#file-handling-since}

* 資料夾會在預先定義的延遲（目前為30秒）後自動更新。
* 其他用戶簽出的檔案被標籤為只讀。
* 檔案會分兩個階段儲存至透過案頭應用程式載入的網路磁碟機位置。
* 在第一階段，檔案會儲存在本機。 如此，儲存檔案的使用者不必等到檔案完全傳輸至AEM後，就可在檔案儲存後立即繼續工作。
* 在第二階段，案頭應用程式會在預先定義的延遲（例如30秒）後，將更新的檔案上傳至AEM伺服器。 此操作在後台進行。 使用「 **顯示背景檔案同步狀態** 」選項可檢視上傳作業的狀態。

## 重要注意事項 {#important-notices}

**資料夾上傳。** 建議您使用新的「資料夾上傳」功能，將較大的階層式資料夾上傳至AEM，而不是使用複製／拖放至從Finder/檔案總管層級載入的AEM儲存庫。 使用檔案夾上傳功能時，案頭應用程式會直接與AEM通訊，因此更能掌控整個程式。

**讓AEM工作階段保持可用。** AEM案頭應用程式需要開啟至AEM Assets伺服器的作業階段，以確保正確運作。 對於每天使用案頭應用程式的使用者，建議在一天結束時卸載AEM Assets以強制登出，然後在早上「裝載AEM Assets」，以確保他們已登入且網路共用可運作。

**在Finder中關閉「圖示預覽」。** 若要使用Finder瀏覽大型資料夾的效能，尤其是在網路連線不佳的情況下，請確定已關閉「圖示」和「圖示預覽」。 否則，Finder會開始下載資料夾中的每個資產，以產生小型預覽，這會導致效能不佳並產生高頻寬使用率(CQ-4219779)

* 在Finder中，前往「AEM Assets共用網路資料夾」
* 按一下右鍵DAM安裝點
* 選擇顯示視圖選項
* 取消選取「顯示圖示預覽」
* 按一下「使用為預設值」

**連線至新的AEM伺服器時清除快取。** 如果案頭應用程式連線至另一個具有相同URL的AEM伺服器，則不會自動清除快取。 手動清除快取，以確保正確運作。 請注意，在測試中通常會發生這種情況，因為在相同URL上執行時，AEM安裝可能會被取代(CQ-4216982)

**使用CA簽署的SSL憑證。** 請注意，當透過HTTPS安全連線連線連線至AEM時，AEM案頭應用程式不支援自簽名SSL憑證。 伺服器上需要CA簽署的憑證才能進行此類連線。 (CQ-87941)

## 已知問題 {#known-issues}

* 一般:
   * 伺服器URL必須指向伺服器，而無路徑(例如 `http://server`、 `https://server`、 `http://server:port`或 `https://server:port`)。 不支援/content/dam以外的上下文路徑和子資料夾(CQ-89343、CQ-87272)
* 檔案名稱／本地化：
   * 未正確處理具有保留字元的檔案和檔案夾名稱。 請確定使用符合AEM需求的檔案和檔案夾名稱(CQ-93361、CQ-93308、CQ-89276、CQ-4217183)
   * 有些應用程式（例如Adobe Illustrator）可能會建立名稱在AEM中不受支援的檔案。 例如，在轉 `Converted` 換檔案後新增，如此就無法上傳檔案(CQ-4216985)
   * 具有國際名稱的資產可能每隔幾秒鐘就會出現並消失
* 登入和登出：
   * 某位使用者簽出的資產無法透過Touch UI的「開啟」動作或直接在案頭上為另一位使用者開啟。 有些應用程式可能會報告為已鎖定，但在嘗試開啟時也會損毀甚至掛起。 (CQ-4199234)
   * 多個使用者同時變更檔案可能會造成部分修改遺失。 因應措施是使用登入和登出功能，防止多位使用者變更相同的檔案(CQ-97035)
   * 某些應用程式不正確支援唯讀標幟，這可讓使用者儲存已由其他使用者簽出的檔案。 在其他用戶簽入檔案之前，不會傳輸修改的檔案。 這兩項修改皆可在AEM中以資產的不同版本取得(CQ-89551、CQ-87572、CQ-89615)
   * Finder會獨立報告已檢出和只讀狀態。 這會在使用者簽出資產時產生2個鎖定圖示(CQ-89507)
* Finder整合：
   * 拖放大型檔案時，當檔案在背景傳輸時，Finder可能會逾時。 這會產生 `Error - 36`。 因應措施是再次拖放或開啟資產(CQ-4219628)
   * 手動資料夾重新載入並不總能運作。 解決方法：  等待30秒，讓資料夾自動更新。 (CQ-97389)
   * 更多資產資訊…… 僅限於單一檔案選擇(CQ-89542、CQ-87656)
   * 在AEM Assets中開啟…… 僅限於單一檔案和檔案夾選擇(CQ-83382)
   * 重新命名沒有副檔名的資產時發生錯誤(CQ-4218971)
* 複製／貼上功能： 當沒有資產複製到剪貼簿時，貼上即可使用
* Windows:
   * 只有NTFS完全支援具有備用資料流(ADS)的檔案。 將這些檔案複製至案頭應用程式提供的WebDAV共用檔案時，會出現警告對話方塊，警告使用者檔案的屬性無法複製到新位置。 這通常不錯，因為屬性僅與使用者案頭上的特定應用程式相關，與實際檔案內容(CQ-103770)(Win)無關
   * Windows版案頭應用程式需要由使用者安裝(CQ-4216389)(win)
   * 在中斷連線後，在某些情況下，當在失敗的上傳上按一下「重試」按鈕後，應用程式可能會當機(CQ-4251884)(Win)
